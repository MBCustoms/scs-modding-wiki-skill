# Security Policy

## Scope

This project is a documentation skill for Claude Code. It contains:

- Markdown documentation files mirrored from the SCS Software Modding Wiki
- JSON index and source mapping files
- PowerShell maintenance scripts
- GitHub community and documentation files

There is no server infrastructure, authentication, or user data handling.

## Supported Versions

| Version | Supported |
|---|---|
| 1.x (current) | Yes |

## Reporting a Vulnerability

If you discover a security issue in this project — for example, a script that performs unsafe operations, or a file that could be weaponized if executed — please report it responsibly.

**Do not open a public GitHub issue for security vulnerabilities.**

Instead, report privately via GitHub's [Security Advisories](https://github.com/MBCustoms/scs-modding-wiki-skill/security/advisories/new) feature.

Include:

- A description of the issue
- The file(s) involved
- Steps to reproduce or a proof of concept
- Your assessment of impact and severity

You will receive a response within 7 days. If the issue is confirmed, a fix will be prioritized and a public advisory published after the fix is released.

## Out of Scope

The following are not in scope for this project's security policy:

- Vulnerabilities in Claude Code itself — report those to [Anthropic](https://www.anthropic.com/security)
- Vulnerabilities in the SCS Software games — report those to [SCS Software](https://www.scssoft.com/)
- Issues with the SCS Modding Wiki content itself
